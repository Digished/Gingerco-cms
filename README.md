# Ginger & Co — CMS

Headless CMS for [gingerandco.at](https://gingerandco.at) — a Vienna-based Afrobeats fitness company. Built on Payload CMS 3 + Next.js 15, deployed on Vercel.

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| CMS & API | [Payload CMS 3](https://payloadcms.com) |
| Framework | Next.js 15 + React 19 + TypeScript |
| Database | PostgreSQL (Supabase or Vercel Postgres) |
| Media storage | Vercel Blob |
| Email | [Resend](https://resend.com) |
| Hosting | Vercel |

## Custom API Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/subscribe` | POST | Accept newsletter signup, trigger double opt-in email |
| `/api/confirm-subscription` | GET | Confirm subscription via token in email link |
| `/api/unsubscribe` | GET | Unsubscribe via token in email link |
| `/api/email-campaigns/:id/send` | POST | Trigger immediate campaign send (admin only) |

## Quick Start

### Prerequisites

- Node.js 20.9+
- PostgreSQL database (local, Supabase, or Vercel Postgres)

### Setup

```bash
cd cms
cp .env.example .env
# Fill in DATABASE_URL and PAYLOAD_SECRET at minimum

npm install
npm run dev
```

Open [http://localhost:3000/admin](http://localhost:3000/admin). On first run Payload prompts you to create an admin user.

## Deployment

### Required Environment Variables

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL connection string |
| `PAYLOAD_SECRET` | Secret for cookie/token encryption (32+ random chars) |
| `RESEND_API_KEY` | Resend API key for all outgoing email |
| `RESEND_FROM_EMAIL` | Verified sender address (e.g. `events@gingerandco.at`) |
| `NEXT_PUBLIC_SERVER_URL` | Public website URL (e.g. `https://gingerandco.at`) |
| `NEXT_PUBLIC_CMS_URL` | CMS deployment URL. **Required when CMS is on a different domain from the main site.** Used to build confirmation/unsubscribe links in emails. |
| `BLOB_READ_WRITE_TOKEN` | Vercel Blob token for media uploads |

### Deploy Steps

1. Import repo at [vercel.com/new](https://vercel.com/new)
2. Set root directory to `cms`
3. Add all environment variables above
4. Deploy — build command is auto-detected: `npm run payload -- migrate && npm run build`

## Email System

Newsletter subscriptions use a double opt-in flow:

1. Visitor submits the **Newsletter Block** form on the website
2. `POST /api/subscribe` creates a `pending` subscriber and sends a confirmation email via Resend
3. Subscriber clicks **"Confirm My Subscription"** in the email
4. `GET /api/confirm-subscription?token=...` sets status to `subscribed` and sends a welcome email
5. Subscribers can unsubscribe at any time via `GET /api/unsubscribe?token=...`

Email campaigns are created in the **Email Campaigns** collection and can be sent immediately via the "Send Campaign" button, or scheduled for a future time.

See `DOCS/PLATFORM_WALKTHROUGH.md` for a full end-to-end guide.
